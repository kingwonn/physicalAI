端侧具身智能大脑:一张图看懂怎么做

跑在机器人机身上的具身智能"大脑"的收敛架构:三层切分(深思 / 反应 / 反射)+ 云边分工 + 算力分档 + 评价标尺 + 模组厂战略读数。数字对准 arXiv/官方规格一手源(完整分析见 GitHub 上的 on-device-brain 页,12 个 section)。

① 收敛架构 · 三层切分

单一模型不能既聪明又快 → 拆成三层,各跑各的时钟、各在各的硬件上。上层想"做什么",下层管"怎么动"。

S2深思 System 2 · VLM5–10 Hz

语义场景/语言理解 + 任务规划。互联网预训练的大 VLM(如 Helix 7B VLM @7–9 Hz;GR00T Eagle-2 VLM @10 Hz)。通用但慢 → 可选云边混合(重推理上云)。

S1反应 System 1 · 视觉运动策略100–200 Hz

把意图变成连续电机指令。小而快的策略(如 Helix 80M @200 Hz 覆盖 35 DoF)。扩散/流匹配动作头比自回归快 ~100x;动作分块(action chunking)开环执行藏延迟。

必须在端侧(on-device)

实时反应环(S1/S0)、安全、断网可用、隐私(入户摄像头)。控制/安全环容不下云往返(µs–ms 确定性)。

可上云 / 云边混合

重推理、大模型规划、车队学习、OTA。移远 Robrain 式"以端为主、云端为辅",断网维持核心功能。5G/MEC 端到端 ~5-10ms(未证实)。

③ "做好"的判据 · 买家/工程师标尺

算力已基本解决;真正卡住的是可靠性与泛化。绿=基本解决 / 黄=部分 / 红=未解决。

✓ 基本解决

延迟

端到端感知→动作;S1 需 100–200 Hz;并行解码/动作分块已达标。VLA-Perf:pi0 在 Thor ~19 Hz。

✓ 基本解决

功耗/散热

TOPS/W 才是指标;Thor 40–130W(占 ~2kWh 包 3–19%);量化 INT4/FP4 + 高能效 SoC(Hailo ~10 TOPS/W)。

✓ 基本解决

成本

算力模组占 BOM ~5–10% 且下降;$/TOPS 竞争;小脑档已白菜价。

◐ 部分

可更新性

OTA + 端侧少样本适配(Gemini On-Device 50–100 示范);但自更新物理机器人的安全性未过关。

◐ 部分

安全

确定性回退 + 安全可停性(CMU/Siemens PRISM);ISO 10218:2025 含延迟/控制频率;分立 MCU vs 片上安全岛未定。

✗ 未解决

泛化

OOD 任务/场景/物体成功率仍差(ACT 见新盘子 86%→~12%);跨本体迁移在进步但不稳。

✗ 未解决

可靠性(9 个 9)

演示 vs 24/7 的部署鸿沟;MTBF 几无公开数据(Spot 巡检 78h MTBI 是少数);评测危机(自报数字、无标准端侧基准)。

④ 安全 + 工程落地(可交付脑模组必答)

功能安全(急停/回退)之外,还有网络安全面和工程落地成本 —— 对"卖脑模组"的军火商定位是真实差异化点。

✗ 薄弱

权重保护 / TEE

整个模型在可被偷的机器人上。Jetson Thor 无机密计算/TDISP(独立 GPU 专属);高通 SPU(EAL4+)可做。需权重加密 + 安全启动(Thor 最多 16 把 OEM 公钥)+ 远程证明。

✗ 未解决

对抗 / 提示注入

RoboPAIR 对 Go2/Jackal 越狱 100% 成功;VLA 对抗补丁物理世界失败率 88.6%;入户机器人可被"哄骗"提示越狱。截至 2026-07 无已发表现场防御。

◐ 前车之鉴

供应链信任 / UniPwn

宇树 G1 蓝牙僵尸网络(UniPwn,CVE-2025-35027 等,硬编码 AES 密钥、可蠕虫到 root);GUARD Act 把连接模组信任推上台面(与移远相关)。安全启动 + 签名 OTA 是底线。

◐ 工程门槛

内存/并发/标定

S2(pi0 ~14GB)+S1+感知+VSLAM+安全监视共驻单 SoC(128GB Thor);单 GPU 分时(MPS 无优先级抢占 → 快环可能被慢 VLM 饿死;需 MIG/优先级)。逐台标定 + bring-up 是 OEM 的真痛点 → 模组厂"预集成大脑"的价值所在。

⑤ 对模组厂(移远式)的战略读数

端侧大脑(算力 + 连接 + 定位模组)是可防御的军火商赛道。不赌哪家整机赢,卖每台机器人都要的三层大脑硬件。

两侧夹击要认清:一边是机电 BOM(减速器/丝杠/电机,占整机 ~50-57%)—— 十年精密加工护城河,模组厂进不去(见零部件供应链);另一边是芯片原厂(高通/NVIDIA)越过模组商直供整机 —— 这才是真威胁,不是富士康的 PIPE。

一个"端侧大脑模组"应该暴露什么(规格清单):

  • 算力分档选项(小脑 10-48 TOPS → 大脑 Thor 级),按机器人类别
  • 实时总线(EtherCAT/CAN-FD)+ 确定性协处理 MCU(服务 S0 层)
  • 连接层(5G/WiFi)用于云边混合 + GNSS 定位(第三条腿)
  • 参考大脑方案(如 Robrain)+ 软件/中间件(ROS2/推理运行时/安全监视/OTA)
  • 补齐短板:峰值算力(77 vs 700/2070 TOPS)、VLA/中间件软件深度、抗原厂去中间化的护城河
