> **说人话:** 这家公司叫光轮智能(LightWheel AI),干的活是给机器人公司"造假数据"——用计算机仿真(在电脑里模拟物理世界,让虚拟机器人练习抓取、搬运)批量生成训练数据,卖给造人形机器人和机器人大模型的公司,客户号称包括 Google DeepMind、Figure、智元、吉利等。它跟我们设计的数据引擎是**同赛道对面路线**:我们押注真机采集(真人戴设备干活录数据),它押注合成数据占未来 90% 以上——它是我们"合成数据威胁"假设的最强现实样本。**身份解析:** 法律实体为光轮智能(北京)科技有限公司,成立于 2023-01-16,注册于北京海淀;另在美国加州 Santa Clara 设有商业实体(PeritasAI 合作通稿电头即 "SANTA CLARA, Calif.",与 Crunchbase、LinkedIn 记录三方独立确认;个别库另载 Cupertino 地址),双总部结构(中国研发+美国 BD);lightwheel.ai 为真实官网(有约 18 篇带日期博客、GitHub 组织、HuggingFace 组织,非抢注空壳)。

## 1. 愿景与论纲 — 逐字引语

- CEO 谢晨:**"5 年以内合成数据的需求量会占到所有数据体量的 90%~95%"**(36氪原文核验);联创杨海波 2025-05 版本更激进:**"未来具身智能合成数据将占比超过 90%,甚至 99% 以上"**(leaderobot.com,2025-05-06 第二届中国人形机器人与具身智能产业大会发言,原文核验)。
- 总裁杨海波,新京报专访:**"具身智能的数据需求量至少是自动驾驶的 1000 倍"**;仿真之所以必需,是因为具身智能需要"感受磁吸阻尼的力"这类精确物理反馈,真实采集无法规模化提供;"未来机器人和智能体会像今天的手机、汽车一样,广泛应用于各个行业和城市角落,背后必然需要可靠的数据支撑。"
- NVIDIA 的 Madison Huang(Omniverse 与 Physical AI 产品市场高级总监,黄仁勋之女)与谢晨同台直播时说:**"语言模型研究者拥有整个互联网的数据,而机器人必须依靠仿真"**——明确把光轮框定为 NVIDIA 需要的"合成数据工厂"(腾讯新闻页面原文核验,仍为单一来源)。
- 值得注意的诚实点:光轮的公开技术叙述承认 Sim2Real gap(仿真到现实的差距,即电脑里练会的动作到真机上不一定管用)只能收窄、不能消灭——其"测量-生成-求解"管线的官方表述是"缩小 Sim2Real gap"(量子位 MEET2026 演讲实录),真实数据仍是校准的地面真值。此前记录的"永远无法被彻底消除"强表述本轮未能定位到原文出处,降级为转述 (unverified as verbatim)。

**批判读法:** 90%~99% 的合成占比预言是卖合成数据的人做出的预言,属于典型的"屁股决定嘴巴";但他们同时承认 gap 只能收窄、真实数据仍是锚,说明其内部技术认知比对外营销口径更克制。**这对我们意味着什么:** 即使按光轮自己最激进的叙事,真实数据也保留了"校准锚"的不可替代地位——我们引擎产出的真机数据恰好卡在这个位置,量小但单位价值高。

## 2. 创始人与团队

- **谢晨 (Dr. Steve Xie),创始人 & CEO。** 背景白话:此前在 NVIDIA(美国)、Cruise(通用汽车旗下自动驾驶公司)、蔚来三家公司从零搭建自动驾驶仿真数据管线,是"合成数据流水线"这件事全球做过最多遍的人之一。北大物理本科,哥伦比亚大学金融学博士(36氪口径;注意:是金融方向博士,不是机器人/CS 博士)。LinkedIn 自述与 36氪、证券时报等多家独立报道交叉确认——此前我们内部标记的"NVIDIA/Cruise/蔚来履历"传言**为真**,非 Morphic 式乌龙。为什么重要:他把自动驾驶行业已验证的仿真数据打法整体平移到具身智能,这是该公司全部论纲的人格化来源。
- **杨海波,联合创始人 & 总裁,现任法定代表人(2025-06 由谢晨变更而来;其 LinkedIn 自署"联合创始人/COO")。** 背景白话:非技术出身——前美团公共事务总监、曾任北京市市领导专职秘书、曾主管政府标准化部门,现兼任北京民合国际交流基金会理事长(百度百科+新京报访谈履历行,交叉确认)。为什么重要:公司二号位的核心资产是政府关系与标准制定通道而非工程能力,与下文"参编 20 项国家/行业标准"的主张形成结构性呼应——这是一种中国市场特有的护城河构建方式。
- **"Jay Yang, Chief Architect"**(NVIDIA 官方案例研究中的受访者)与杨海波是否同一人**未确认,不应混同** (unverified)。
- 团队(公司自述):成员来自 NVIDIA、Cruise、蔚来、百度、华为、达摩院,清北毕业生为主;曾宣称要建"千人仿真运营团队"——这是 2025-05 的愿景表述,非当前实际人数 (unverified)。团队基因是自动驾驶仿真系,而非游戏引擎/影视特效系,与 Unity/Unreal 背景的竞品形成区隔。

**这对我们意味着什么:** "AV 仿真老兵 + 政府事务操盘手"的组合意味着光轮同时打技术牌和标准牌;我们若与其正面竞争,拼的不是仿真技术,而是它天然做不了的真机物理接触数据。

## 4. 技术栈方案

- **物理/仿真底座:** NVIDIA Isaac Sim + Isaac Lab + Omniverse(OpenUSD)。白话:全部押在英伟达的仿真全家桶上。为什么重要:光轮与 **Newton 物理引擎**(NVIDIA/DeepMind/Disney 合作的下一代引擎,见 [simulation](simulation.md))深度绑定——新浪财经报道称其"接受谷歌 DeepMind、英伟达和 Disney Research 的联合邀请,作为核心指导委员加入开源 GPU 加速物理引擎 Newton",其 GitHub 组织下亦有公开的 Newton-Lightwheel 仓库佐证——这不是普通客户关系,是生态共建位,竞品拿钱也买不到。
- **数据生成方法:** MimicGen + DexMimicGen(白话:把一条人类遥操作示范自动"繁殖"成 100–1000 条变体)叠加 SimReady 资产;Cosmos 世界模型做轨迹生成;"测量-生成-求解"框架——先物理测量真实物体,再生成视觉与物理都匹配的仿真资产,最后 GPU 求解动力学。为什么重要:这是把 gap 收窄工程化的完整闭环,不是纯 AIGC 生图。
- **开源足迹(真实,已核验,与多数空壳同行形成鲜明对比):** GitHub 组织 LightwheelAI 共 16 仓,含 LW-BenchHub(268 任务的可复现评测枢纽,leisaac 仓约 692 星)、LeIsaac(基于 SO101Leader/LeRobot 的 IsaacLab 遥操作工具)、Lightwheel-simready-asset 开源资产仓("251 物体 + 8 环境"细分数字本轮未在仓库页复核 (unverified));LW-BenchHub 数据集经 README 实证含 **219 独立任务、21,500 条示范 / 20,537,015 帧**(HuggingFace 按 Double-Piper/X7S/G1-WBC/G1-Controller 等 4 个本体分包发布;平台层宣称支持 7 类机器人 27 个变体);另有与学术团队 OpenDriveLab 联名的 LightwheelOcc 自动驾驶占据栅格数据集(托管于 OpenDriveLab 名下)——说明 AV 血统仍是活的产品线。
- **论文与专利:** 本轮检索**未发现** arXiv 署名论文,**未发现** CNIPA 专利记录(见 §6)。
- **标准参与:** 宣称"主导或参编 20 项国家、行业相关标准"(数据质量/仿真平台/工业数据集)——单一来源(ZAKER 聚合稿),未在工信部/SAC 注册库直接核到 (unverified),但与杨海波的标准化履历结构自洽。

**这对我们意味着什么:** 光轮的开源披露密度按我们 [data-companies-compare](data-companies-compare.md) 的评分尺可能超过现榜首 GenRobot(3/5),我们对"合成数据商=黑箱"的刻板判断需要更新;但其栈 100% 绑定 NVIDIA,意味着任何去 NVIDIA 化的客户(或受出口管制影响的客户)是它的天然盲区。

## 7. 与我们设计的对撞要点

1. **对撞我们的"真机数据不可替代"公理**([data-engine-first-principles](data-engine-first-principles.md) 第一性推导:接触物理与失败分布必须来自真实世界):光轮自己的官方表述只承诺"缩小 Sim2Real gap"而非消灭它,且真实测量数据是其管线的校准锚——**对手的自认恰好划定了我们的安全区**:高接触、高失败代价、长尾物理(布料/流体/力控)场景的真机数据是合成管线的输入而非输出,我们应把引擎产出显性定位为"Sim2Real 校准锚数据",甚至可以卖给光轮这类公司。
2. **对撞 [data-engine-v2](data-engine-v2.md) 的采集设备选型:** EgoSuite 的设备无关策略(Aria/Quest/Vision Pro/GoPro 通吃)与我们 v2 的低成本自研采集头思路相反——它用软件层抹平硬件差异,我们用硬件一致性换标定精度。若光轮的多设备对齐管线真能工作,我们"统一硬件才能保数据质量"的前提需要压力测试;反之,我们的一致性硬件是 EgoSuite 做不到的力/触觉通道载体。
3. **对撞我们的"数据质量>数据数量"下注:** 光轮 HF 上 2053 万帧公开数据 + MimicGen 百倍繁殖,走的是"量大管饱+评测层自证"路线;我们 v2 押注单条高保真(带力反馈、带失败标注)。RoboFinals 若成为行业事实考试院,**数据的价值将由它的考卷定义**——我们必须尽早让自家数据在 LIBERO/RoboCasa 系基准上跑出可引用分数,否则质量主张无法定价。
4. **对撞觅蜂案例**([case-mifeng](case-mifeng.md)):智元既是光轮客户(NVIDIA 案例页)又孵化了竞品觅蜂,谢晨还曾在觅蜂发布会同台——头部玩家在合成与真采两条线同时下注、竞合不分。这支持我们 v2 的判断:数据买方会多源采购,单一路线排他不成立,我们无需"打败"合成路线,只需在它的短板通道上成为默认供应商。
5. **对撞我们对合成阵营的"弱护城河"旧判**(data-collection-devices.md 曾判:合成数据易复制、gap 是买方问题):光轮的 Newton 核心指导委员身份(新浪报道+GitHub 仓库佐证)、NVIDIA 案例页背书、真实开源仓库、以及(若属实的)20 项标准参编,构成生态位+规制型护城河,**不是**"数据易复制"能概括的。我们的竞争分析需从"合成数据没壁垒"升级为"壁垒在生态绑定而非数据本身"——同理,我们的壁垒也应建在客户工作流绑定上,而非数据文件本身。
